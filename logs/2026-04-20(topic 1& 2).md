## Variables & Memory Model

Before today I thought:
→ values are stored in a variable directly
→ mutable means we can change values, immutable means we can't

What I now know:
→ A variable is not a box, it is a label
→ The actual value like 10, "hi", [1,2,3] is called an object
→ When we do x = 10, Python creates object in memory
   and points label x to that object location
→ id() tells us exact memory location of the object
→ Reference count goes UP when we assign same object to new label
→ Reference count goes DOWN when a label moves away from object
→ Garbage collector runs when reference count hits 0 — object deleted

## Mutable vs Immutable

Immutable means:
→ object cannot be changed in memory
→ Python creates a NEW object every time
→ Examples: int, str, float, tuple, bool
→ Safe to share same memory location because value can never change

Mutable means:
→ object CAN be changed in memory
→ same object gets modified at same location
→ Examples: list, dict, set
→ Never shares location because value can change — sharing is dangerous

## The Bug I Learned — Default Mutable Argument:
→ Bug happens because default [] is created once when Python
  loads the function — same mutable object reused every call
  — all values keep adding to same list — application behaves wrongly
→ Fix: use None as default, create fresh list inside if statement
  so every call gets its own new list

## 💻 Code I Ran Today

# Reference counting
import sys
x = 10000
y = x
z = x
print(sys.getrefcount(x))  # 6 — x, y, z + internals
y = "hi"
print(sys.getrefcount(x))  # 5 — y left
z = "hi"
print(sys.getrefcount(x))  # 4 — z left

# Mutable vs Immutable proof
x = [1,2,3]
print(id(x))
x.append(4)
print(id(x))   # same — list is mutable

x = 10
print(id(x))
x = 20
print(id(x))   # different — int is immutable

# Sharing proof
a = (1,2,3)
b = (1,2,3)
print(id(a) == id(b))  # True — immutable, safe to share

a = [1,2,3]
b = [1,2,3]
print(id(a) == id(b))  # False — mutable, never shares

## ⚠️ Gotchas
1. sys.getrefcount() always shows higher count than expected
   — it adds its own temporary reference + interpreter internals
   — focus on the CHANGE not the absolute number
2. Default mutable argument bug — classic Python trap
   — never use [] or {} as default argument

## 🎤 Interview Angle
Junior answer: "variable stores a value"
Senior answer: "variable is a label pointing to an object in memory.
               Python uses reference counting to track how many
               labels point to an object. When count hits 0,
               garbage collector deletes the object automatically."
